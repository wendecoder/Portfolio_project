# API Reference

## Getting Started
- Base URL: This project is not deployed yet. Currently, it runs only on a local host. The backend is hosted on the default,` http://127.0.0.1:5000`, which is set as a proxy in the frontend configuration.
- Authentication: It doesn't require authentication or API keys.

## Error Handling
The errors from the API are returned as JSON objects in the following format:
```
	{
            "success": False,
            "message": "Resource not found",
            "error": 404
        }
```
There are five error types that is returned by the API when a request fail. These are:
- 400, Bad request
- 404, Resource not found
- 405, Method not allowed
- 422, Unprocessable

## Endpoints

`GET '/categories'`

- Fetches a dictionary of categories in which the keys are the ids and the value is the corresponding string of the category
- Request Arguments: None
- Returns: An object that have a key, categories, that contains an object of id: category_string key:value pairs, and success to show the request was successful.
`Sample: curl http://127.0.0.1:5000/categories `
```
	{
  "categories": {
    "1": "Science", 
    "2": "Art", 
    "3": "Geography", 
    "4": "History", 
    "5": "Entertainment", 
    "6": "Sports"
  }, 
  "success": true
	}
```

`POST '/quizzes'`

- Sends a post request in order to get the next question
- Request argument: Nothing
- Returns: a single new question object

`Sample: curl -X POST -H "Content-Type: application/json" -d '{"previous_questions": [1, 2, 20, 15], "quiz_category": {"type": "Science", "id": 1}}' http://127.0.0.1:5000/quizzes`

```
{

    "answer": "Blood", 
    "category": 1, 
    "difficulty": 4, 
    "id": 22, 
    "question": "Hematology is a branch of medicine involving the study of what?"
  }, 
  "success": true
}
```
